# Rim Medical — متجر المعدات الطبية 🏥

موقع متكامل لمتجر معدات طبية، يتكون من:

| الجزء | المسار | الوصف |
|---|---|---|
| موقع الزبائن | `index.html` | موقع عرض فاخر ثلاثي اللغات (عربي / فرنسي / إنجليزي) |
| لوحة التحكم | `admin.html` | إدارة المنتجات والأرقام والشعار — محمية بكلمة مرور |
| البيانات | `data.js` | "قاعدة البيانات" الحالية (منتجات + إعدادات) |
| طبقة البيانات | `store.js` | الوسيط بين الصفحات والبيانات (هنا تتم الترقية لاحقاً) |

**كلمة مرور لوحة التحكم:** هي الكلمة التي حددتَها (غير مكتوبة هنا حفاظاً على السرية —
تُخزَّن في الكود كبصمة مشفرة فقط). يمكنك تغييرها في أي وقت من تبويب "أرقام التواصل".

**ما يمكنك إدارته من اللوحة:** المنتجات (بعدة صور لكل منتج)، الفئات (إضافة/تعديل/حذف)،
رقما واتساب، الإحصائيات، روابط السوشيال، والشعار.

---

## 1) المعاينة محلياً على جهازك

أبسط طريقة: انقر نقراً مزدوجاً على `index.html` وسيفتح في المتصفح.

الطريقة الأفضل (تضمن عمل كل شيء 100% وربط اللوحة بالموقع):

```bash
# إذا كان لديك Python (موجود غالباً في كل الأجهزة):
python3 -m http.server 8000
# ثم افتح في المتصفح:
# الموقع:        http://localhost:8000
# لوحة التحكم:   http://localhost:8000/admin.html
```

## 2) كيف يرتبط الموقع بلوحة التحكم؟ (اقرأ هذا جيداً)

النظام الحالي يعمل **بدون خادم وبدون قاعدة بيانات** — وهذا مقصود لتبدأ بأبسط حل:

1. البيانات الافتراضية (المنتجات، الأرقام، الشعار) موجودة في ملف `data.js`.
2. عندما تعدّل شيئاً من لوحة التحكم، يُحفَظ التعديل في **متصفحك أنت** (localStorage)
   ويظهر فوراً في موقع الزبائن **على جهازك أنت** (جرّب: افتح الموقع واللوحة في تبويبين
   وعدّل رقماً — سيتغير في الموقع مباشرة).
3. **لكي يرى كل الزوار تعديلاتك**، اذهب إلى تبويب **"النشر والنسخ الاحتياطي"** واضغط
   **"تنزيل ملف البيانات (data.js)"**، ثم ارفع الملف الذي نزل مكان `data.js`
   في موقعك (الخطوات بالأسفل). لا تحتاج لتعديل أي سطر كود — مجرد استبدال ملف.

> لماذا هكذا؟ لأن استضافة الملفات الثابتة مجانية 100% ولا تحتاج أي إعدادات.
> عندما يكبر المتجر، رقِّ النظام إلى Supabase (الشرح في القسم 4) فتصبح التعديلات
> تلقائية تماماً بدون خطوة الرفع.

## 3) النشر المجاني خطوة بخطوة (GitHub Pages)

الموقعان (الزبائن + اللوحة) يُنشران معاً لأنهما في نفس المجلد.

1. أنشئ حساباً في [github.com](https://github.com) إن لم يكن لديك.
2. أنشئ مستودعاً جديداً (Repository) باسم مثل `rim-medical` واجعله **Public**.
3. ارفع كل ملفات هذا المجلد إليه (زر **Add file → Upload files** يسمح بالسحب والإفلات).
4. من صفحة المستودع: **Settings → Pages → Branch: main → Save**.
5. بعد دقيقة أو دقيقتين سيكون موقعك متاحاً على:
   - الموقع: `https://اسم-حسابك.github.io/rim-medical/`
   - لوحة التحكم: `https://اسم-حسابك.github.io/rim-medical/admin.html`

### تحديث المحتوى لاحقاً (بدون لمس الكود)
1. عدّل ما تريد من لوحة التحكم (منتجات، أرقام، شعار).
2. تبويب "النشر" → **تنزيل ملف البيانات (data.js)**.
3. في GitHub: اضغط على ملف `data.js` في الصفحة الرئيسية للمستودع → أيقونة القلم ✏️ (أو
   Upload files لرفع الملف الجديد فوقه) → **Commit changes**.
4. خلال دقيقة تظهر التغييرات لكل الزوار. هذا كل شيء.

بدائل مجانية أخرى تعمل بنفس الطريقة: [Netlify](https://netlify.com) (سحب وإفلات
المجلد كاملاً) أو [Vercel](https://vercel.com).

## 4) الترقية لاحقاً إلى قاعدة بيانات حقيقية (Supabase)

عندما تريد أن تنعكس التعديلات **تلقائياً** بدون خطوة "تنزيل/رفع الملف"، وأن تديـر
المتجر من أي جهاز:

### لماذا Supabase؟
مجاني (خطة سخية)، أسهل من Firebase للمبتدئين، ويوفر: قاعدة بيانات + تخزين صور +
نظام تسجيل دخول حقيقي.

### الخطوات العامة
1. أنشئ حساباً في [supabase.com](https://supabase.com) → **New Project** (مجاني).
2. من **Table Editor** أنشئ جدولين:
   - `products`: أعمدة `id (text)`, `category (text)`, `name (jsonb)`, `desc (jsonb)`,
     `price (numeric, nullable)`, `image (text)`, `icon (text)`, `badge (text, nullable)`
   - `settings`: صف واحد فيه `customer_whatsapp`, `institutional_whatsapp`, `logo`, `stats (jsonb)`, `social (jsonb)`
3. من **Storage** أنشئ Bucket عام باسم `images` لصور المنتجات والشعار
   (بدل تخزينها كنص Base64).
4. من **Authentication** فعّل الدخول بالبريد الإلكتروني، وأنشئ حساباً لنفسك —
   هذا يستبدل كلمة مرور اللوحة الحالية بحماية حقيقية.
5. فعّل **Row Level Security**: القراءة للجميع، والكتابة للمستخدم المسجَّل فقط.
6. **التعديل البرمجي الوحيد المطلوب**: ملف `store.js` هو الوسيط الوحيد بين
   الصفحات والبيانات، وكل صفحات المشروع تستدعي `RimStore.getData()` و
   `RimStore.saveData()` فقط. أي مطور (أو أنا في جلسة قادمة) يستبدل داخل هذا الملف
   القراءة من localStorage بالقراءة من Supabase — **بدون لمس صفحات الموقع أو اللوحة
   إطلاقاً**.

النتيجة بعد الترقية: تحفظ من اللوحة → يُكتب في Supabase → يظهر لكل الزوار فوراً.

## 5) ملاحظات مهمة

- **الأمان:** كلمة مرور اللوحة الحالية حماية على مستوى المتصفح فقط (تمنع الوصول
  العرضي). لا تضع في اللوحة بيانات حساسة، والحماية الحقيقية تأتي مع ترقية Supabase
  (الخطوة 4 أعلاه). لوحة التحكم لا تُفهرس في محركات البحث (`noindex`).
- **أرقام واتساب:** تُكتب بالصيغة الدولية بدون `+` (مثال: `22228646987`).
  رقم الزبائن ورقم المؤسسات منفصلان تماماً ويُعدَّلان من تبويب "أرقام التواصل".
- **الصور:** تُضغَط تلقائياً عند الرفع من اللوحة. إذا ظهرت رسالة "مساحة التخزين
  ممتلئة" فهذا حد localStorage (~5MB) — استخدم صوراً أقل أو ارفع روابط صور، أو
  رقِّ إلى Supabase Storage.
- **إضافة لغة رابعة:** كل النصوص في كائن `I18N` أعلى سكربت `index.html` —
  انسخ قسم `en` وترجمه.
- **آراء الزبائن:** نصوص تجريبية (Placeholder) داخل كائن `I18N` قسم `reviews`،
  عدّلها بآراء حقيقية عندما تتوفر.
